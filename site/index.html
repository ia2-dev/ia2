Information Architecture for Intelligent Agents

The web should be legible to agents without becoming theirs.

IA² explores how documents and applications can publish explicit meaning where people already see and use information. Agents can inspect it, validate it, and follow it without hidden semantic payloads or platform-specific integrations.

Read the HTML/RDF model ↗ Try the live demo

HTML is the first proving ground, not the boundary of the project.

01 / Premise

One information surface.

The web already contains our broadest information architecture, but its machine-facing layer is fragmented. Meaning is inferred from presentation, isolated behind APIs, or shipped in payloads detached from the content a person sees.

IA² keeps presentation and machine-readable meaning directly correlated. An inert document can be downloaded and inspected. A live application can expose its current semantic state. Both can remain independent of the agent that reads them.

02 / Architectural horizon

A legibility layer for more than pages.

Each layer answers a different question. Keeping them separate is what makes the whole architecture composable.

  1. 01

    Surfaces and bindings

    Where does the information live, and how does it relate to what a person sees?

    HTML and live DOM today; more host-native bindings later.
  2. 02

    Semantic models

    What does the information mean?

    Full RDF 1.2, open vocabularies, profiles, and mappings.
  3. 03

    Integrity and evidence

    Why should a consumer accept, qualify, or revisit it?

    SHACL, provenance, confidence, validation, and signatures.
  4. 04

    Agency

    What could be done, under which conditions, and by whose authority?

    Inspectable affordances with execution kept separate.
  5. 05

    Navigation and exchange

    How does information move without losing identity or context?

    Discovery, fragments, snapshots, queries, diffs, and subscriptions.

03 / Inspect what exists

The first vertical slice is already navigable.

  • 01 HTML/RDF core model An attribute binding for complete RDF 1.2 datasets. Specification ↗
  • 02 Live semantic workspace An issue tracker and inbox whose RDF changes with the DOM. Interactive demo ↗
  • 03 HTML RDF Navigator Extraction, navigation, and serialization tied back to DOM carriers. TypeScript package ↗

04 / North star

Agents should be able to come to independently published information and understand it on its own terms.

The result is not one universal agent API. It is a web of addressable, composable information that many agents and applications can navigate without translating everything into a platform-owned schema.
